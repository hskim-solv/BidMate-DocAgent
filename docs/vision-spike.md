# Layout-aware visual ingestion spike (Donut)

Tracks issue #168. Honest 1-page comparison of the OCR baseline (`pymupdf` text layer + `pytesseract` fallback) against a layout-aware vision encoder-decoder (`Donut`). Closes the "no layout-aware vision foundation model evaluated" gap called out in [`visual_ingestion.py`](../visual_ingestion.py)'s module docstring (lines 4-10).

## Scope

- **Single 1-page synthetic PDF** with title / metadata / fields / table / paragraph (generated reproducibly by the runner; no committed binary).
- **Two pipelines compared**: current `pymupdf+pytesseract` vs. Donut on the rendered page image.
- **Decision documented here, not enforced** — the OCR pipeline remains the default per ADR 0001's preserve-baseline invariant. Donut is opt-in via `BIDMATE_VISUAL_OCR=donut` env var or `get_ocr_provider("donut")`.

## Runner

```bash
# Both pipelines on a generated synthetic PDF, write results into this doc
pip install torch transformers sentencepiece     # one-time, ~800MB+
python3 scripts/run_donut_spike.py --write-doc

# Use a real (e.g. private) Korean RFP PDF — no ground truth, side-by-side dump
python3 scripts/run_donut_spike.py --input path/to/rfp.pdf

# Just the baseline (no torch needed)
python3 scripts/run_donut_spike.py --backend baseline
```

Optional model override:

```bash
BIDMATE_DONUT_MODEL=naver-clova-ix/donut-base python3 scripts/run_donut_spike.py
```

Default model is `daekeun-ml/donut-base-finetuned-korean` (Korean-finetuned per issue recommendation) with automatic fallback to `naver-clova-ix/donut-base` (English-trained generic). The runner caches the model in-process so repeated `--input` runs don't reload.

## Requirements & gotchas

- `torch` is **not** in `requirements.txt` — installed only when running the spike. Default install stays lean.
- **`torch>=2.6` required** — older versions reject `.bin` checkpoint loading per [CVE-2025-32434](https://nvd.nist.gov/vuln/detail/CVE-2025-32434). Donut models still ship `.bin` (not `.safetensors`).
- **`sentencepiece`** is required by Donut's tokenizer; install it explicitly.
- The Korean-finetuned model ID `daekeun-ml/donut-base-finetuned-korean` may be unavailable on HF (404). The runner falls back to the base model on load failure and labels the result with the actually-loaded model.
- **GPU strongly recommended.** CPU inference takes seconds per page; the spike is only practical at scale on GPU.

## Decision

**Keep the OCR baseline as the default.** Donut adoption is gated on the following criteria, none yet met:

1. Measured ≥X percentage-point lift on the **private 100-doc RFP corpus** (not synthetic), per [ADR 0005](adr/0005-eval-split-public-synthetic-private-local.md)'s real-data discipline.
2. GPU available in the production / CI path (CPU latency is prohibitive).
3. Stable Korean-finetuned model checkpoint with `.safetensors` weights (avoids the torch≥2.6 friction).
4. ADR proposed and accepted to displace the extractive OCR path per [ADR 0001](adr/0001-preserve-naive-baseline.md)'s preserve-baseline invariant.

For now, Donut is a **diagnostic option** the reviewer can switch on for exploration:

```python
from visual_ingestion import get_ocr_provider, parse_visual_document
parse_visual_document(pdf, ocr_provider=get_ocr_provider("donut"))
```

## Reproduce / extend

Both pipelines are pluggable via the `OcrProvider` interface ([`visual_ingestion.py:44`](../visual_ingestion.py)). Adding another vision model (e.g. `pix2struct`, ColPali, LayoutLMv3) is a new provider function returning either `str` or `list[dict[text, bbox, confidence]]` — the existing `normalize_ocr_result` ([`visual_ingestion.py:577-581`](../visual_ingestion.py)) handles both shapes.

The regression test [`tests/test_visual_donut_regression.py`](../tests/test_visual_donut_regression.py) guards the wiring without loading any model; CI runs it as part of `bash scripts/test.sh`.
